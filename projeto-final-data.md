# Projeto Final — Módulo 7 (Data Science)

> Projeto integrador de Data Science end-to-end — aluno escolhe o problema.

## Informações do Módulo

- **Aulas:** 6 (de 4h cada)
- **Carga horária:** 24h
- **Posição na trilha:** [Data Science#fase 3 Projeto Final](../cursos/data-science#fase-3--projeto-final.md)

---
## Aulas
### Aula 1 — Kickoff: Definição de Problema e Dados

**Conteúdo (fixo — todos passam por aqui):**
- Revisão de Git e GitHub para Data Science
- Estrutura de projeto DS profissional (cookiecutter-data-science)
- Definição do problema de negócio: pergunta que o projeto responde
- Identificação e coleta do dataset (Kaggle, APIs, dados abertos)
- EDA inicial: tamanho, qualidade, viabilidade
- GitHub Projects: board com etapas do projeto
- Planejamento de entregáveis por aula
- Alinhamento de expectativas com professor

**Exercício prático:**
1. Repositório criado com estrutura profissional, README com definição do problema
2. Dataset identificado e primeira EDA documentada
3. GitHub Projects com etapas planejadas

**Recursos:**
- [Cookiecutter Data Science](https://drivendata.github.io/cookiecutter-data-science/)
- [Kaggle Datasets](https://www.kaggle.com/datasets)
- [Google Dataset Search](https://datasetsearch.research.google/)

**Tempo:** 4h (1h revisão + 1h GitHub Projects + 2h planejamento e EDA inicial)

---

### Aula 2 — EDA e Preparação de Dados

**Conteúdo (fixo — todos passam por aqui):**
- EDA sistemático do dataset escolhido
- Limpeza: nulos, duplicatas, outliers
- Feature engineering específico para o problema
- Visualizações exploratórias
- Documentação das decisões de limpeza
- Divisão treino/teste (se projeto de ML)
- Pipeline de pré-processamento
- Versionar dados processados no DVC ou S3

**Exercício prático:**
1. Notebook de EDA completo commited no GitHub
2. Dataset limpo e processado pronto para modelagem

**Tempo:** 4h (3h hands-on + 1h revisão)

---

### Aula 3 — Tira-Dúvidas: Modelagem e Análise

**Conteúdo (planejado — ajustar conforme projeto):**

**Se projeto de ML supervisionado:**
- Baseline model
- Seleção de algoritmos
- Treinamento e avaliação
- Tuning de hiperparâmetros
- Feature importance

**Se projeto de análise/BI:**
- Análises estatísticas avançadas
- Visualizações narrativas
- Segmentação ou clustering
- Insights acionáveis

**Se projeto de séries temporais:**
- Decomposição e sazonalidade
- Modelos de forecast
- Avaliação de forecast

**Exercício prático:**
1. Modelo ou análise central do projeto implementado e avaliado

**Tempo:** 4h (30min teoria específica + 3h projeto + 30min revisão)

---

### Aula 4 — Tira-Dúvidas: Deploy e Visualização

**Conteúdo (planejado — ajustar conforme projeto):**

**Se deploy de modelo:**
- API FastAPI de predição
- Docker + deploy no Render ou AWS
- Documentação da API

**Se dashboard:**
- Dashboard Dash ou Power BI
- Publicação e compartilhamento
- Storytelling com os dados

**Se relatório/análise:**
- Notebook polido como documento final
- Slides de apresentação
- Publicação no Kaggle ou GitHub

**Exercício prático:**
1. Deploy ou dashboard funcionando e acessível publicamente

**Tempo:** 4h (30min teoria específica + 3h projeto + 30min revisão)

---

### Aula 5 — Tira-Dúvidas: Polish e Storytelling

**Conteúdo (planejado — ajustar conforme projeto):**
- Refinamento do modelo ou análise
- Storytelling: criar narrativa coerente com os dados
- Visualizações finais profissionais
- README completo com contexto e resultados
- Slide deck de apresentação (5-7 slides)
- Preparação para o pitch
- Publicação pública (GitHub, Kaggle, LinkedIn)

**Exercício prático:**
1. Projeto completo e polido, pronto para apresentação
2. Slide deck de 5-7 slides preparado

**Tempo:** 4h (30min teoria específica + 3h projeto + 30min ensaio de apresentação)

---

### Aula 6 — Apresentação Final

**Conteúdo (fixo — todos apresentam):**
- Cada aluno apresenta o projeto (7-10 minutos)
- Demo ao vivo do modelo ou dashboard
- Apresentação de slides com insights
- Perguntas da turma
- Celebração! 🎉

**Formato de apresentação:**
1. Problema de negócio e por que importa (1min)
2. Dados e abordagem (1min)
3. Insights principais (3min) — slides ou demo
4. Deploy ou entregável final (1min)
5. Limitações e próximos passos (30s)
6. Perguntas (2min)

**Critérios de avaliação (sugestão):**
1. Problema bem definido e relevante
2. Dados coletados, limpos e documentados
3. Análise ou modelo com avaliação adequada
4. Visualizações claras e informativas
5. Entregável público (API, dashboard ou notebook)
6. Apresentação clara dos insights

**Tempo:** 4h (apresentações de 7-10min por aluno)

## Projetos Sugeridos

1. Previsão de churn de clientes (classificação)
2. Análise completa do Olist (e-commerce brasileiro)
3. Preço de imóveis: modelo preditivo + insights (regressão)
4. Análise de sentimentos de reviews de app
5. Forecast de demanda de produto (séries temporais)
6. Dashboard de saúde pública (dados DATASUS ou similar)
7. Análise socioeconômica de municípios brasileiros
8. Detecção de fraude em transações

## Requisitos Mínimos

9. Dataset real com pelo menos 1000 registros
10. EDA documentado no Jupyter
11. Modelo ou análise central implementada e avaliada
12. Visualizações profissionais (pelo menos 5)
13. Entregável público (API, dashboard ou notebook no Kaggle/GitHub)
14. README com contexto, metodologia e resultados
15. Slide deck de apresentação
