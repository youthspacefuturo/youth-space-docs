- **Carga horária:** 32h

## Aulas
### Aula 1 — UX: O Básico e Research

**Conteúdo:**
- O que é UX (User Experience) e qual a diferença para UI
- A importância do usuário no centro do processo
- O que é UI (User Interface) — a parte visual
- Research: identificando o público-alvo
- Personas: criando perfis fictícios de usuários
- Jobs to be Done (JTBD): o que o usuário realmente quer accomplishing
- Análise competitiva: estudando produtos similares
- Benchmark: o que funciona e o que não funciona

**Exercícios práticos (escolha um):**
1. Criar 3 personas detalhadas para um app de delivery de comida (jovem universitário, mãe trabalhadora, idoso)
2. Fazer research de 3 apps de fitness e listar: o que têm em comum, o que funciona bem, o que poderia melhorar
3. Desenvolver JTBD para uma plataforma de cursos online (3 jobs principais do aluno, 2 do professor)

**Recursos:**
- [Interaction Design Foundation - UX Design](https://www.interaction-design.org/literature/topics/ux-design)
- [HubSpot Make My Persona](https://www.hubspot.com/make-my-persona)

**Tarefas de casa (escolha uma ou mais):**
1. Criar personas para um app de Finanças Pessoais (3 perfis diferentes)
2. Pesquisar e documentar 5 maus exemplos de UX (interfaces difíceis de usar) e explicar por que falham
3. Entrevistar 2 pessoas sobre um app que usam diariamente, identificar pontos de dor e desejos
4. Mapear o journey completo de um usuário em um processo que você usa frequentemente
5. Criar um relatório de análise competitiva de 3 produtos similares ao seu projeto final

**Tempo:** 4h (1h teoria + 2h prática + 1h discussão)

---

### Aula 2 — UX: Wireframing e User Flow

**Conteúdo:**
- O que é wireframe e quando usar
- Baixa fidelidade vs média vs alta fidelidade
- Princípios de hierarquia visual (o que é mais importante)
- Wireframing no papel vs ferramentas digitais
- Ferramentas: Figma, Balsamiq, Whimsical
- User Flow: mapeando o caminho do usuário
- Fluxos comuns: login, cadastro, compra, busca
- Mapas de site (sitemap) e arquitetura de informação
- Navegação: o que todo usuário precisa encontrar em 3 cliques

**Exercícios práticos (escolha um):**
1. Wireframe de baixa fidelidade no papel: tela de login, tela inicial, tela de perfil — 3 telas do projeto final
2. Criar User Flow completo: desde o primeiro acesso até a conclusão de uma compra em um e-commerce
3. Wireframe digital no Figma: tela principal de um app de tarefas, com header, lista e FAB (botão de ação flutuante)

**Recursos:**
- [Figma - Getting Started](https://help.figma.com/)
- [Whimsical](https://whimsical.com/) — wireframes e user flows
- [Maze](https://maze.co/) — testar wireframes

**Tarefas de casa (escolha uma ou mais):**
1. Wireframe completo de um app de delivery: 8-10 telas cobrindo todo o fluxo (home → busca → produto → carrinho → checkout → confirmação)
2. Mapear User Flow de 3 funcionalidades do Instagram (curtir, comentar, Stories)
3. Criar sitemap de um site de receitas com pelo menos 15 páginas
4. Wireframe de dashboard: tela inicial de um admin com métricas, gráficos e tabela de dados
5. Desenvolver 2 versões de wireframe para a mesma tela (teste A/B conceitual)

**Tempo:** 4h (1h teoria + 2h prática + 1h apresentação)

### Aula 3 — UX: Princípios Heurísticos e Acessibilidade

**Conteúdo:**
- 10 Heurísticas de Nielsen para usabilidade
- Visibilidade do status do sistema
- Correspondência entre sistema e mundo real
- Controle e liberdade do usuário (undo/redo)
- Consistência e padrões
- Prevenção de erros
- Reconhecimento em vez de memorização
- Flexibilidade e eficiência de uso
- Design estético e minimalista
- Recuperação de erros
- Ajuda e documentação
- Acessibilidade: o que é e por que importa
- WCAG: níveis A, AA, AAA
- Contraste de cores e tamanho de fonte
- Navegação por teclado e leitores de tela
- HTML semântico como base

**Exercícios práticos (escolha um):**
1. Auditar uma interface existente (site do governo ou app) usando as 10 heurísticas e dar nota de 1 a 5 para cada
2. Identificar 5 barreiras de acessibilidade em um site e propor soluções para cada uma
3. Criar um checklist de usabilidade e acessibilidade para revisar antes de entregar um projeto

**Recursos:**
- [Nielsen Norman Group - Heuristics](https://www.nngroup.com/articles/ten-usability-heuristics/)
- [WebAIM - Contrast Checker](https://webaim.org/resources/contrastchecker/)
- [A11Y Project Checklist](https://www.a11yproject.com/checklist/)
- [Lighthouse - Accessibility Audit](https://developer.chrome.com/docs/lighthouse/accessibility/)

**Tarefas de casa (escolha uma ou mais):**
1. Realizar auditoria completa de acessibilidade no site de uma empresa local (comprovação de contraste, navegação, labels)
2. Criar um documento "Antes e Depois" de um site problemático, mostrando as correções de acessibilidade aplicadas
3. Testar um site com leitor de tela (NVDA no Windows ou VoiceOver no Mac) e documentar a experiência
4. Listar 10 apps que você considera com boa usabilidade e explicar qual heurística cada um aplica bem
5. Criar uma apresentação curta (5 slides) explicando o que é acessibilidade digital e por que é importante

**Tempo:** 4h (1h teoria + 2h prática de auditoria + 1h discussão)

### Aula 4 — Figma: Interface e Primeiros Passos

**Conteúdo:**
- Interface do Figma: canvas, toolbar, painel de camadas
- Frames vs Groups: quando usar cada um
- Criando shapes: retângulos, elipses, linhas, polígonos
- Ferramentas de texto: tipografia, alinhamento, line height
- Cores: preenchimento, borda, gradientes
- Auto Layout: o que é e por que muda tudo
- Spacing e padding com Auto Layout
- Criando componentes reutilizáveis
- Variants: componentes com estados
- Assets e ícones: encontrando e usando
- Organizando o arquivo: páginas e nomenclatura

**Exercícios práticos (escolha um):**
1. Recriar o design de um card de produto do Nubank (ou outro app que admire) no Figma, usando Auto Layout
2. Criar um botão com 4 estados: default, hover, pressed, disabled — usando Variants
3. Montar uma tela de login com: logo, inputs (email, senha), botão primário, link "esqueci a senha" — tudo com Auto Layout

**Recursos:**
- [Figma Community](https://figma.com/community) — arquivos gratuitos para estudo
- [Figma for Beginners](https://help.figma.com/)
- [DesignCode - Figma Tutorial](https://designcode.io/figma)

**Tarefas de casa (escolha uma ou mais):**
1. Recriar 5 componentes do iOS (botões, switches, inputs) no Figma usando Auto Layout
2. Desenvolver um set de 10 ícones customizados (usando formas simples) para um app fictício
3. Criar uma tela de onboarding com 3 steps usando Variants e Auto Layout
4. Organizar um arquivo Figma caótico (pastas, nomenclatura, páginas) em estrutura profissional
5. Desenhar um perfil de rede social no Figma: avatar, nome, bio, posts — usando Auto Layout

**Tempo:** 4h (1h teoria + 2h prática + 1h peer review)

### Aula 5 — Figma: Sistema de Design

**Conteúdo:**
- Design System: o que é e por que criar
- Cores: escala de cores, semântica (primária, secundária, status)
- Tipografia: escala modular (8pt grid), títulos e corpo
- Espaçamento: escala consistente (4, 8, 12, 16, 24, 32, 48...)
- Componentes do sistema: Botão, Input, Card, Badge, Modal
- Atomic Design: átomos, moléculas, organismos
- Documentando o Design System
- Atomic Design na prática: Botão (átomo) → Grupo de Inputs (molécula) → Formulário (organismo)
- Mantendo consistência em projetos grandes
- Versionamento do Design System

**Exercícios práticos (escolha um):**
1. Criar um Design System básico: 5 cores, 4 tamanhos de texto, 5 espaçamentos, 3 componentes (botão, input, card)
2. Expandir um componente: criar botão com 3 variantes de cor (primary, secondary, ghost) e 2 tamanhos (small, large)
3. Mapear uma tela em Atomic Design: identificar átomos, moléculas e organismos presentes

**Recursos:**
- [Design Systems Repo](https://designsystemsrepo.com/)
- [Material Design 3](https://m3.material.io/) — referência
- [Figma - Design Systems](https://www.figma.com/community/design-systems)

**Tarefas de casa (escolha uma ou mais):**
1. Criar um Design System completo para o projeto final: cores, tipografia, espaçamentos, componentes
2. Documentar todos os componentes do Figma em um arquivo separado com guidelines de uso
3. Implementar um "Token" system: todas as cores e espaçamentos como variáveis nomeadas semanticamente
4. Criar um "Kit de Sobrevivência": componentes essenciais (15-20) necessários para montar qualquer tela
5. Estudar o Design System do Spotify, Slack ou Airbnb e documentar as principais decisões

**Tempo:** 4h (1h teoria + 2h prática + 1h apresentação)

### Aula 6 — Figma: Prototipagem e Interações

**Conteúdo:**
- Prototyping no Figma: Frames como telas
- Smart Animate: transições automáticas
- Trigger: on tap, on hover, while pressing
- Actions: navigate to, open URL, swap component, toggle
- Easing: ease-in, ease-out, spring
- Prototype interactions: menu hamburger, accordion, tabs
- Creating flows: connectando telas
- Preview do protótipo no Figma
- Exportando como vídeo/GIF
- Testando no dispositivo com Figma Mirror

**Exercícios práticos (escolha um):**
1. Protótipo de login: tela de login → loading → home (com Smart Animate)
2. Menu hamburger: clique abre menu com animação deslizante da esquerda
3. Tabs: 3 abas com conteúdo diferente, transições suaves entre elas

**Recursos:**
- [Figma - Prototyping](https://help.figma.com/hc/en-us/articles/360040314193)
- [Figma - Smart Animate](https://help.figma.com/hc/en-us/articles/360040537395)
- [Dribbble](https://dribbble.com/) — inspiração de UI

**Tarefas de casa (escolha uma ou mais):**
1. Criar protótipo completo do app To-Do com: criar tarefa, completar, deletar, filtrar — todo o fluxo
2. Desenvolver protótipo de e-commerce: home → produto → carrinho → checkout → confirmação
3. Criar micro-interações: like button com animação, dropdown com transição, FAB que vira toolbar
4. Prototipar onboarding de 5 telas com transições progressivas
5. Recriar a interação de uma interface que você admira (ex: o Stories do Instagram)

**Tempo:** 4h (1h teoria + 2h prática + 1h teste em grupo)

### Aula 7 — Figma: Design Responsivo e Entrega para Dev

**Conteúdo:**
- Design responsivo no Figma: constraints e resizing
- Criando múltiplas variações de tela (desktop, tablet, mobile)
- Componentes adaptativos com Auto Layout
- Frame overlay para mobile preview
- Plugins úteis: Iconify, Content Reel, Remove BG
- Exportando assets: SVG, PNG, @1x, @2x, @3x
- Specs: gerando CSS/Flutter/Swift para devs
- Redlining: documentando espaçamentos e fontes para devs
- Handoff: processo de entrega para desenvolvimento
- Ferramentas de handoff: Figma Inspect, Zeplin
- Comunicação designer-desenvolvedor: o que facilitar o trabalho

**Exercícios práticos (escolha um):**
1. Criar design responsivo de um dashboard: versão desktop (3 colunas), tablet (2 colunas), mobile (1 coluna)
2. Exportar todos os ícones de um projeto como SVG e documentar uso em SPECS
3. Criar redline document: especificar todos os espaçamentos, cores e fontes de uma tela para o dev

**Recursos:**
- [Figma - Resizing](https://help.figma.com/hc/en-us/articles/360040570406)
- [Iconify](https://iconify.design/) — ícones para todos os frameworks
- [Figma Inspect](https://help.figma.com/hc/en-us/articles/360055785053)

**Tarefas de casa (escolha uma ou mais):**
1. Desenvolver versão mobile de um app de delivery: todas as telas adaptadas para 375px
2. Criar um "Redline Guide" completo de um componente, especificando cada pixel para o dev
3. Preparar handover package: arquivo organizado, pages com dev notes, components catalog
4. Usar o plugin "HTML CSS Export" para gerar código real de um componente criado
5. Simular uma reunião de handover: explicar as decisões de design para um "dev" (colega de turma)

**Tempo:** 4h (1h teoria + 2h prática + 1h role-play)

### Aula 8 — Projeto: Design Completo do Projeto Final

**Conteúdo:**
- Revisando todo o conteúdo de UX e UI
- Aplicando UX Research ao projeto final (personas, user flows)
- Design de todas as telas no Figma
- Criando Design System próprio
- Implementando protótipo navegável
- Testando com usuários (mesmo que colegas)
- Iterando baseado no feedback
- Preparando entrega final
- Apresentação do design ao "cliente"

**Exercício prático:**
1. Design completo do projeto final: research, wireframes, Design System, telas de alta fidelidade, protótipo interativo

**Sugestões de entrega:**
1. App de delivery: todas as telas do fluxo (8-12)
2. Dashboard de analytics: telas de overview, detalhamento, filtros
3. Rede social para gamers: perfil, feed, chat, ranking

**Desafio extra (para rápidos):**
1. Criar Design System completo com documentação no Figma
2. Gravar vídeo de walkthrough do protótipo como se fosse uma apresentação de produto
3. Escrever um "UX Case Study" documentando o processo completo

**Recursos:**
- [Behance](https://www.behance.net/) — portfolio inspiration
- [Dribbble](https://dribbble.com/) — UI inspiration
- [UX Collective - Medium](https://uxdesign.cc/)

**Tarefas de casa (escolha uma ou mais):**
1. Fazer upload do projeto no Figma Community (visibilidade profissional)
2. Criar perfil no Behance ou Dribbble com o case study completo do projeto
3. Preparar pitch de 3 minutos: explicar as decisões de UX/UI para a turma
4. Escrever Medium article documentando o processo: "Como diseñé meu primeiro app"
5. Testar o protótipo com 2 usuários reais (colegas ou amigos) e documentar o feedback

**Tempo:** 4h (30min revisão + 3h projeto + 30min apresentação)

## Projeto do Módulo

Ao final do módulo, o aluno entrega:

1. Research documentado (personas, user flows)
2. Wireframes de baixa/média fidelidade
3. Design System completo (cores, tipografia, espaçamentos, componentes)
4. Telas de alta fidelidade no Figma
5. Protótipo interativo navegável
6. Teste com pelo menos 1 usuário real
7. Feedback incorporado ao design final
